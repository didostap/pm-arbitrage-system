## Challenges & Learnings

### 1. Story 7.5.2 Was Too Large
- **Challenge:** 18 tasks spanning persistence layer, service layer, controller, enrichment service, Prisma migration, AND full frontend page with 7 sections. Most code review findings of any Epic 7.5 story (7 findings: 2C, 2H, 2M, 1L).
- **Root cause:** Scope grew organically â€” "while we're in there, add balance, add SL/TP projections..." Each addition felt small but the aggregate was a 4-layer beast.
- **Resolution:** New rule: if a story spans more than two architectural layers (persistence, service, controller, frontend), it gets split. No exceptions without documented approval from architect.
- **Key insight (Arbi):** "Task count is a symptom; layer sprawl is the disease. A 14-task story that stays within service + controller is fine. A 9-task story that touches persistence, service, controller, and frontend is the problem."

### 2. Story Specs Drifting from Codebase Reality
- **Challenge:** Three distinct spec-to-codebase mismatches:
  - Epic referenced `GET /api/risk/state` which doesn't exist (resolved by extending `GET /dashboard/overview`)
  - 7.5.3 spec said rate limit codes 1003-1005; actual codebase uses 1002 and 1009
  - 7.5.3 spec said `currentPnl`; DTO exposes `unrealizedPnl`
- **Root cause:** Story author writing against architecture docs or PRD instead of actual codebase. With 40+ completed stories, the codebase has diverged significantly from original planning artifacts.
- **Resolution:** Pre-story codebase grep checkpoint: every AC referencing an endpoint, field name, error code, or DTO is verified against the actual codebase before the story is marked ready-for-dev. Formalized as a gate in the story creation workflow.
- **Key insight (Arbi):** "40+ stories in, the codebase IS the spec."

### 3. Broken Prisma Migration Directory
- **Challenge:** `20260307021134_add_knowledge_base_resolution_fields` had an empty directory with no `migration.sql`. Discovered during 7.5.2 when trying to create a new migration. Required reconstruction from DB introspection and manual checksum update.
- **Root cause:** Likely a failed `prisma migrate dev` that created the directory but didn't write the SQL file. Dormant until the next migration attempt.
- **Resolution:** New DoD gate: after every `prisma migrate dev`, verify the migration SQL file is non-empty before committing.

### 4. API Client Regeneration Requires Running Server
- **Challenge:** `swagger-typescript-api` requires a running server to fetch the Swagger JSON. Both 7.5.2 and 7.5.3 hand-edited the generated `Api.ts` file instead of regenerating.
- **Root cause:** No static Swagger JSON export in the build pipeline.
- **Resolution:** 2-hour spike in Sprint 0.8 to test consuming a static JSON export. If unsolved, document manual process as official workaround.

## Key Insights

1. **Pattern reuse compounds** â€” single utility/interface serving multiple consumers across stories; Prior Story Intelligence sections enable seamless handoff

4. **No telescoping** â€” get the data model right first, then layer behavior on top with empirical data; applies directly to Epic 8's schema-then-scoring sequence

7. **Fire-and-forget with batch completion events** â€” reusable pattern for long-running operator actions
