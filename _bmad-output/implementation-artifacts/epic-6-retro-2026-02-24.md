## Challenges & Learnings

### 1. Financial Math Violations in "Display" Code
- **Challenge:** Story 6.1 code review found native JS subtraction in formatters (`fillPrice - price` for slippage, `currentValue - threshold` for breach amount). Fixed to `decimal.js`.
- **Root cause:** Formatters feel like string templates — mental model is "building a message" not "doing arithmetic." But any subtraction of two prices IS financial math regardless of where the code lives.
- **Resolution:** Absolute rule — any arithmetic on a field that touches money uses `decimal.js`. No context-based exceptions. No judgment calls about "display code." Added to gotchas.md and story template.
- **Action:** Charlie to grep full codebase for native arithmetic on monetary fields in Phase 0.

### 2. Event Payload Gaps — Designed for Alerting, Used for Persistence
- **Challenge:** `OrderFilledEvent` lacks `contractId`, `fees`, `gas`. CSV trade log has 5 N/A columns. Tax report had similar gaps (`positionId`, `correlationId`, `proceeds` all N/A or misleading).
- **Root cause:** Events designed in Epic 5 for operator alerting. Epic 6 uses them for CSV persistence and audit trail — different consumers with different data needs.
- **Resolution:** Document gap inventory now (Dana). Batch enrichment in one Epic 8 story rather than patching events across modules piecemeal. Dashboard (Epic 7) will identify additional gaps.
- **Decision (Arbi):** Don't refactor events now. Note missing fields during Epic 7 consumption. Single enrichment pass in Epic 8.

### 3. Tax Report P&L Uses expectedEdge Proxy
- **Challenge:** `OpenPosition` model lacks `realizedPnl` column. Tax report uses `expectedEdge` split 50/50 between platforms as P&L proxy.
- **Root cause:** Schema designed for position tracking, not for tax reporting. `realizedPnl` requires tracking actual fill prices on both legs through exit.
- **Resolution:** Documented limitation with CSV disclaimer. Schema migration for `realizedPnl` deferred to Phase 1.

### 4. Code Review Fix Clustering — Patterns, Not Random
- **Analysis (Dana):** 25 fixes clustered around: financial math violations (2), error hierarchy compliance (2), data model gaps in exports (5), missing test coverage (2), dead code (2), documentation (3), correctness/wiring (6), cosmetic (3).
- **Insight:** Financial math and data model gaps are systemic — they need process fixes, not "be more careful." Error hierarchy and test coverage are improving (fewer violations per story over time).
