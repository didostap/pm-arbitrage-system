- Live paper trading data exposed a display bug (exit proximity showing 100% when should be ~0%)

- Constraint: can't fetch prices in async event handlers without blocking fan-out

- Avoids stale P&L in WS payloads racing against fresh P&L on next REST poll

## Challenges & Learnings

### 1. Story 7.1 Was Too Large
- **Challenge:** 9 tasks, 3 parts (backend, frontend, Docker), 5 debug incidents, 24 code review fixes. More debugging than previous two epics combined.
- **Root cause:** First-contact story with multiple new architectural layers (React SPA, WebSocket gateway, nginx, Swagger codegen, Docker entrypoint) combined into one vertical slice.
- **Resolution:** Split first-contact stories at the *integration boundary*, not the feature boundary. Separate "backend API surface" story (gateway, Swagger, DTOs, Docker) from "frontend scaffold and connect" story. Integration bugs isolated to second story with stable backend already in place.
- **Key insight (Arbi):** "The lesson isn't 'vertical slices are bad' - it's 'first-contact stories with a new architectural layer should be split at the integration boundary, not the feature boundary.'"

### 2. Swagger Spec Was Untested Contract for Six Epics
- **Challenge:** All 10 existing endpoints across 5 controllers had void/any types in generated client. Response wrapper DTOs missing. CLI plugin overrides conflicting with manual decorators.
- **Root cause:** Swagger decorators were decorative - no programmatic consumer until dashboard needed a typed client. Latent debt accumulated across Epics 1-6, surfaced all at once in 7.1.
- **Resolution:** CLI plugin inverts maintenance burden (code drives spec, not reverse). Long-term: CI-generated client as tested contract to catch drift incrementally.
- **Key insight (Arbi):** "We should have caught this at the architecture level. The architecture doc specifies a typed client from Swagger, but we never enforced that contract until Epic 7."

### 3. Error Severity Drifts Without Structural Defaults
- **Challenge:** `severity: 'error'` instead of `'critical'` (7.1 H8/M8). Hardcoded severity in controller responses (7.3 M4). Two stories, same category of bug.
- **Root cause:** Severity is set at the call site as a string, no enum enforcement, no default-per-class. Every `throw new SystemHealthError(...)` requires the developer to choose correctly.
- **Resolution:** Severity should be a property of the error class with a sensible default mapped by code range, overridable at construction. Structural gate instead of reviewer discipline.

### 4. First-Contact Infrastructure Gotchas
- **Challenge:** nginx `add_header` doesn't inherit into child `location` blocks (H5). CSP `connect-src` allowing `ws://localhost:*` in production (H6). Unescaped env vars in shell-to-JS injection (H7).
- **Root cause:** New infrastructure layers (nginx, Docker entrypoint) have their own gotcha categories that backend developers don't carry as intuition.
- **Resolution:** Treat new infrastructure layers like unfamiliar frameworks - spike-first or dedicated focused story.

## Key Insights

1. **Module boundary discipline pays off at scale** - six epics of architectural rules enabled a full-stack dashboard to bolt on without touching the hot path
2. **Split first-contact stories at the integration boundary** - smaller blast radius, faster debug cycles, more focused code reviews
3. **Constraints beat vigilance** - structural gates (error class defaults, CI-generated clients, CLI plugin) remove bug categories; reviewer discipline catches stragglers
4. **Earned confidence, not complacency** - 8/8 retro follow-through in a favorable environment; Epic 8 is the real stress test
5. **Start boring, escalate with evidence** - applies to algorithms (string heuristics before NLP), infrastructure (native APIs before libraries), and scope (lookup table before learning system)
6. **Untested contracts accumulate latent debt** - enforce programmatic consumers early
7. **Code review earns its time cost beyond correctness** - performance optimizations, security hardening, architectural shortcuts caught before production
8. **Validation loops work** - paper trading infra (Epic 5.5) -> validation (Epic 6.5) -> real bug caught and fixed (Story 7.5)

3. **Architectural decision: semantic scanning cadence** - Document in story spec. Separate service, startup-time or operator-triggered, writes to ContractMatch. NOT wired into detection hot path. Completely decoupled.

## Retrospective Commitments Summary

### What We Learned
- Module boundary discipline creates systems where major new capabilities bolt on without disruption
- First-contact stories with new architectural layers need integration-boundary splitting
- Untested contracts (Swagger spec) accumulate latent debt that surfaces painfully at first consumption
- Structural constraints compound; vigilance-dependent rules decay
- Favorable environments inflate follow-through metrics - the real test is the next unfamiliar domain

### What We'll Do Differently
- **Split first-contact stories at integration boundary** - backend API surface separate from frontend integration
- **Error severity as class property** - structural default per code range, optional override
- **Spike unfamiliar problem domains** - not just unfamiliar frameworks
- **Start boring** - string heuristics, lookup tables, simple before sophisticated
- **Timebox spikes with negative findings as valid outcomes** - 4-hour hard stop, no separation = finding

### What We'll Keep Doing

- Story-to-story intelligence handoff (Previous Story Intelligence sections)
